# Теория. Sequential Vs Random Access

CPU любит предсказуемый доступ к памяти.

Последовательный проход хорош тем, что:

- cache lines используются полностью;
- hardware prefetcher может заранее подтягивать следующие линии;
- TLB/cache работают устойчивее;
- loop легко оптимизировать.

Random access хуже:

- каждая загрузка может идти в далекое место памяти;
- cache line часто используется только частично;
- hardware prefetcher не видит простого pattern;
- latency памяти начинает доминировать над арифметикой.

В low latency коде часто выигрывает не "меньше C++ строк", а более предсказуемый
memory access pattern.
