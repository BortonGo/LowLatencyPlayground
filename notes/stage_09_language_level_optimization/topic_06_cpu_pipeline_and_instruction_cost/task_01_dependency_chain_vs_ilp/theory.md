# Теория. Dependency Chain Vs ILP

CPU может выполнять несколько независимых инструкций параллельно, если между
ними нет зависимости по данным.

Один accumulator:

```cpp
sum += values[i];
```

создает цепочку:

```text
sum_next зависит от sum_previous
```

Следующее сложение не может полностью завершиться независимо от предыдущего.

Несколько accumulator-ов разбивают одну длинную цепочку на несколько коротких:

```cpp
acc0 += ...
acc1 += ...
acc2 += ...
acc3 += ...
```

Так CPU получает больше independent work и может лучше заполнить pipeline.

Это не про количество C++ строк, а про data dependencies.
